1. 文件範圍與優先規則

文件目的： 本文件為 GGAP 遊戲聚合後台最終版交付規格，供 PM、RD、QA、營運、財務、風控與後端 API 串接使用。 若舊版文件或模組文字與本文件衝突，均以本文件為準。
項目最終規格
產品定位 B2B 遊戲聚合平台後台，支援管理者、代理、商戶三種入口，集中管理供應商、遊戲、商戶、代理、會員、交易、帳務、品質與系統設定。
不包含內容 不描述演示前台、不描述 Demo 視覺畫面、不記錄臨時前端展示方案；僅保留正式系統邏輯與技術規格。
正式帳務幣別 settlement_currency = USDT。所有正式對帳、日結、帳單、代理應收、供應商應付與平台毛利皆以 USDT 為準。
顯示幣別 display_currency 只供商戶前台、會員幣別錢包、營運報表與明細參考，不作正式跨幣別結算依據。
商戶定位 商戶是真正接入 API、會員、錢包、遊戲權限與投注資料的營運主體；但不是 GGAP 平台正式收款主體。
代理定位 代理是 GGAP 平台正式應收與結算主體。代理底下可有下級代理與商戶，最多三級代理。
供應商定位 供應商是 GGAP 平台自行處理的成本與應付主體，不與代理或商戶形成直接帳務關係。

2. 最終定版核心決策

決策定版內容影響範圍
供應商結算拆分 供應商結算不可依 agent_id 作主維度；正式維度為 provider_id + settlement_currency + period，可選加 game_type 或 provider_rate_version。 財務中心、供應商帳務、帳單、資料表、API。
代理結算主體 平台只對代理正式收款。代理結算維度為 agent_id + settlement_currency + period，明細可展開商戶、下級代理、供應商與顯示幣別。 代理帳務、代理帳單、下級代理結算、商戶結算。
商戶不作平台正式帳務主體 系統可計算商戶應向代理繳交的金額，但此資訊應歸在代理帳務內的「商戶結算」，不是新增獨立平台商戶帳務。 商戶管理、代理帳務、報表。
代理最多三級 代理層級固定為 L1 / L2 / L3。L1 可新增 L2，L2 可新增 L3，L3 不可再新增下級代理。正式對 GGAP 結算可歸集到 L1 或平台直營代理。 代理管理、資料權限、代理後台、帳務彙總。
費率位置 供應商成本費率放在供應商管理；代理費率放在代理管理；商戶報價放在商戶綁定代理或商戶設定內。三者可依供應商覆寫，未覆寫則使用預設費率。 供應商管理、代理管理、商戶管理、代理帳務、平台毛利。
錢包模式 供應商側只認 GGAP 單一 USDT 錢包；商戶側可使用 Seamless Wallet 或 Transfer Wallet；Transfer Wallet 由 GGAP 內部帳本處理。 Wallet Router、交易中心、會員管理、轉點紀錄。
匯率與日結 每日北京時間 00:00 產生公告匯率；01:00 結算 T-1 帳期。日結後匯率快照、結算金額與帳單來源鎖定。 匯率管理、日結、報表、帳單、交易快照。
重要修正： 舊文件中若出現「結算維度：agent_id + provider_id + settlement_currency + period」應拆成兩種： 供應商結算使用 provider_id + settlement_currency + period； 代理結算使用 agent_id + settlement_currency + period，其中 provider_id 只作代理費率套用與明細分析，不代表代理與供應商直接掛鉤。

3.2 兩條正式財務線

財務線對象主維度說明
平台 ↔ 供應商 供應商應付 provider_id + settlement_currency + period 平台根據供應商報表、平台 Provider 交易快照、供應商合約與成本費率計算應付供應商金額。
平台 ↔ 代理 代理應收 agent_id + settlement_currency + period 平台根據代理樹底下商戶產生的 GGR、代理費率、調帳與服務費計算代理應收。
代理內部 下級代理結算、商戶結算 parent_agent_id + child_agent_id / merchant_id + period 用於算出 L1 應向下級代理或商戶收多少；屬代理帳務展開，不形成 GGAP 對商戶正式應收。
平台內部 平台毛利 period + provider_id + agent_id + settlement_currency 把代理收入線與供應商成本線拉在一起作管理分析，非正式對外帳單主體。

3.3 利潤來源

來源公式備註
供應商與代理差額 agent_receivable - provider_cost 例如供應商成本 4%，平台給代理 7%，差額 3% 為主要毛利來源。
匯率服務費 max(base_usdt_amount, 0) × exchange_fee_rate MVP 預設 0.5%。只對正向應收計算，負 GGR 不收取。
調帳與其他成本 adjustment_amount - activity_cost - other_cost 活動成本、人工調帳、補償、特殊合約都需進入平台毛利分析。

3.4 核心公式

公式說明
display_ggr = display_bet_amount - display_win_amount原幣別 GGR，只作明細、商戶報表與營運參考。
base_usdt_amount = display_ggr ÷ base_ratebase_rate 方向固定為 1 USDT = N display currency。
service_fee_amount = max(base_usdt_amount, 0) × exchange_fee_rate匯率服務費僅對正向金額計算。
final_settlement_usdt = base_usdt_amount + service_fee_amount日結後正式 USDT 金額。
agent_receivable = max(final_settlement_usdt, 0) × agent_rate + adjustmentGGAP 對代理的正式應收。
merchant_receivable_to_agent = max(final_settlement_usdt, 0) × merchant_quote_rate + adjustment商戶應向代理繳交的金額，屬代理帳務明細。
provider_payable = provider_ggr_usdt × provider_cost_rate + fixed_fee + adjustment供應商正式應付，應依供應商報表或 Provider 交易快照計算，不直接由代理日結產生。
platform_margin = agent_receivable - provider_payable + exchange_service_fee - adjustment - activity_cost平台毛利管理報表。

6. 模組規格

6.1 商戶管理

項目規格
定位 商戶是真正串接 API、管理會員、錢包模式、遊戲權限與投注資料的營運主體；所有商戶必須綁定代理。
新增流程 基本資料 → 代理歸屬 → API 設定 → 錢包與幣別 → 遊戲權限 → 商戶報價 → 確認送出。
主要欄位 merchant_id、merchant_code、merchant_name、agent_id、agent_path、wallet_mode、display_currencies、settlement_currency、callback_url、ip_whitelist、status。
錢包 支援 Seamless Wallet 與 Transfer Wallet。Transfer Wallet 需有內部帳本與會員轉點紀錄。
商戶報價 綁定代理時帶入代理費率，可依每個供應商設定商戶報價；未指定則使用統一服務費率或商戶預設加成。
操作 查看、編輯、啟用、停用、凍結、重置 API Secret、測試 Callback、設定遊戲權限、轉移代理、建立商戶後台帳號、查看操作紀錄。

6.2 代理管理

項目規格
層級最多三級：L1 / L2 / L3。L3 不可新增代理。需防止循環引用與跨樹移動。
正式收款GGAP 正式對代理收款。若 L2 / L3 存在，系統可將正式收款對象歸集到 L1 或平台直營代理，並在代理帳務中展開下級代理結算。
新增流程基本資料 → 上級代理 → 代理層級 → 可用供應商 → 費率設定 → 帳號權限 → 確認。
費率代理建立時設定預設費率；可依每個供應商覆寫。供應商成本只作參考與毛利試算，不代表代理與供應商直接結算。
操作查看、編輯、啟用、停用、凍結、建立下級代理、綁定商戶、設定費率、查看代理帳務、查看下級代理結算、查看商戶結算。

6.3 供應商管理

項目規格
定位供應商是平台成本帳主體，由 GGAP 自行對帳與付款，不直接掛代理或商戶。
新增流程基本資料 → API 設定 → 遊戲同步 → 交易設定 → 幣別語系 → 地區限制 → 成本費率 → 測試環境。
成本費率接入供應商時即可填入成本費率，可依遊戲類型、供應商費率版本、生效時間設定。
負 GGR可設定保留到下期或清零，並需與供應商合約版本一起保存。
維護供應商與遊戲都需支援一次性、每週、每月維護排程；維護期間影響 Launch 與遊戲可見狀態。
操作查看、編輯、啟用、停用、維護、同步遊戲、測試 API、查看供應商帳務、查看操作紀錄。

6.4 遊戲管理與遊戲分組

模組規格
遊戲管理 管理遊戲主檔、供應商、類型、RTP、支援幣別語系、上下架、維護排程、商戶授權、地區限制。維護支援固定每週或每月。
遊戲分組 建立遊戲包、類型分組、供應商分組、商戶專屬分組與活動分組；分組只影響展示與授權，不改變 Provider API 或正式交易流水。
獎池與活動 MVP 可先保留資料結構與入口；正式活動成本需能進平台毛利。

6.5 交易中心

模組規格
會員管理會員錢包唯一鍵為 merchant_id + merchant_player_id + display_currency。商戶需自行依幣別切分會員錢包。
注單管理查詢 Provider Round、注單 ID、交易 ID、會員、商戶、代理、Provider、遊戲、Wallet 模式、雙幣別金額與狀態。
交易流水保存 Bet、Win、Rollback、Refund、Transfer In、Transfer Out 等交易；必須保存 idempotency key、request/response、trace_id。
Wallet Router依商戶 wallet_mode 分流 Seamless Callback 或 Transfer Wallet Ledger。Provider 不知道商戶錢包模式。
轉點紀錄Transfer Wallet 必須有會員轉入、轉出、投注扣款、派彩加款、回滾與餘額變化紀錄。
補單 / 重送處理 Provider Timeout、Merchant Callback Timeout、狀態不一致、Rollback 找不到原交易等異常。

6.6 品質中心

模組規格
風控管理設定商戶、會員、遊戲、金額、頻率、IP、設備、異常贏分、負餘額等規則。
監控告警監控 Launch、Provider、Wallet Callback、交易失敗率、延遲、匯率異常、日結失敗、批次失敗。
操作紀錄所有高風險操作需記錄 operator、role、portal、module、before/after、reason、ip、trace_id。

6.7 系統管理與開發者中心

模組規格
開發者中心放置 API 文件、簽名規則、Callback 範例、錯誤碼、Sandbox 測試、Callback 測試、IP 白名單與 Secret 重置入口。
系統設定管理時區、幣別、匯率來源、告警門檻、交易重試、日結時間、資料保留、匯出限制。
人員與權限支援功能權限、資料權限、角色、職級、2FA、登入安全與審核流程。

7. 費率與商戶報價規格

7.1 費率層級

層級設定位置用途可否依供應商覆寫
供應商成本費率 供應商管理 平台計算供應商成本與平台毛利。 可，並可依遊戲類型或版本。
代理費率 代理管理 平台向代理收款的費率。新增代理時即可決定給此代理的價格。 可，未指定則使用代理預設費率。
商戶報價 商戶管理 / 綁定代理 代理要向商戶收多少。系統只作代理帳務內的商戶結算，不形成平台對商戶應收。 可，未指定則使用商戶預設加成或統一服務費率。
匯率服務費率 匯率管理 / 代理設定 多顯示幣別換算 USDT 時另計的結算服務費，MVP 預設 0.5%。 可依代理或合約調整。

8. 財務中心規格

8.1 財務中心模組

模組主體主要內容
供應商帳務 供應商 供應商成本、供應商對帳、供應商應付帳單、供應商付款紀錄。不得以代理或商戶作正式帳務主體。
代理帳務 代理 代理應收、日結明細、帳單、收款紀錄、帳務差異、下級代理結算、商戶結算。
平台毛利 平台內部 代理應收、供應商成本、匯率服務費、調帳、活動成本、毛利試算。
匯率管理 幣別 幣別設定、每日匯率、服務費率、交易匯率快照、匯率異常告警。
帳單管理 代理 / 供應商 代理應收帳單、供應商應付帳單。使用 target_type + target_id 避免主體混淆。
調帳紀錄 帳期 / 對象 所有人工補正、費率修正、差異處理與帳務調整，需保留原因與審核紀錄。

8.2 代理帳務內部頁籤

頁籤內容
基本資料帳期、正式收款代理、代理層級、結算幣別、狀態、應收摘要。
日結明細依日期、供應商、商戶、顯示幣別展開，用於套用不同代理費率。
帳單代理應收帳單、付款狀態、付款證明、核銷狀態。
收款紀錄代理付款、入帳、沖帳與未收金額。
帳務差異代理確認差異、調帳、補單、Carry Forward。
下級代理結算計算 L1 對 L2、L2 對 L3 的應收或分潤，不形成供應商帳。
商戶結算計算代理應向商戶收多少，作為代理報表與帳務依據。
操作紀錄帳務產生、確認、調整、開帳、鎖帳、收款與匯出紀錄。

11. 批次、冪等與一致性

11.1 每日匯率批次

  1. 每日北京時間 00:00 執行。
  2. 讀取啟用中的 display_currency。
  3. 呼叫第三方匯率 API，產生 exchange_rate_daily。
  4. 若 API 失敗，重試；仍失敗則沿用上一筆有效匯率並告警。
  5. 波動超過門檻需進入待確認，不可自動覆蓋已鎖定帳期。

11.2 每日日結批次

  1. 每日北京時間 01:00 執行，帳期為 T-1 00:00:00 至 23:59:59。
  2. 先建立 daily_trade_summary，依代理、商戶、供應商、顯示幣別彙總交易。
  3. 產生 agent_settlement，作為代理應收與下級代理 / 商戶結算來源。
  4. 產生或更新平台毛利分析資料。
  5. 供應商正式帳單不可直接由代理日結來源產生，需由 provider_settlement 根據 Provider 交易快照、供應商報表與供應商費率計算。
  6. 日結完成後鎖定 batch，後續差異需走調帳或下一期修正。

11.3 冪等規範

場景唯一鍵規則
Provider 交易provider_id + provider_transaction_id重送時返回原結果，不重複扣加款。
商戶 Callbackmerchant_id + idempotency_key + transaction_type保存 request / response。
Transfer Walletaccount_id + idempotency_key餘額更新需使用交易鎖或樂觀鎖。
日結批次batch_id + period + dimension同一帳期同一維度不得產生多筆生效結算。
調帳adjustment_id不得直接改歷史交易，只能新增調帳、補單或 reversal。

12. 安全、稽核與驗收

12.1 安全規格

項目規格
API 安全HMAC-SHA256 或 RSA 簽名、時間戳、防重放、IP 白名單、API Key / Secret。
敏感資料Password、API Secret、2FA Secret、Token 不可明文儲存或寫入 Log，只可遮罩或重置。
高風險操作匯率覆蓋、費率異動、補單、重送、調帳、鎖帳、重置 Secret 需輸入原因，可要求 2FA 或審核。
資料權限MASTER 全域；AGENT 僅代理樹；MERCHANT 僅自身商戶。後端必須以 token claims 強制限制。
金額精度DB 使用 DECIMAL 或整數最小單位；API 金額建議以字串傳輸；不可用 float / double 處理財務。

12.2 驗收清單

類別驗收項目
帳務模型供應商帳務未掛代理或商戶作為正式主體；代理帳務包含下級代理與商戶結算展開。
代理層級L1 / L2 / L3 正確限制，L3 不可新增代理，代理樹資料不可跨權限查詢。
商戶新增商戶必須綁定代理；代理入口只能在自身代理樹新增商戶；商戶報價可依供應商設定。
供應商管理供應商接入時可設定成本費率與負 GGR 政策；供應商結算維度正確。
交易流水每筆交易保存 display_currency、display_amount、settlement_currency、settlement_amount、exchange_rate_id。
Transfer WalletTransfer In / Out、Bet、Win、Rollback 均產生 ledger，before / after balance 正確。
匯率批次00:00 產生公告匯率；異常重試與告警；日結不因後續匯率更新重算。
日結批次01:00 依 T-1 匯率產生交易快照、代理日結與平台毛利分析；供應商正式帳務獨立產生。
API 串接所有列表支援分頁、排序、篩選；所有回應包含 code、message、data、trace_id。
稽核所有費率、匯率、補單、調帳、鎖帳與 Secret 操作具 before/after、reason、operator、trace_id。
最終結論： GGAP 正式模型為「供應商成本帳」與「代理收入帳」完全拆分； 商戶是真正接入與營運主體，但商戶應收只作代理帳務展開； 平台毛利才把代理應收、供應商成本、匯率服務費與調帳合併分析。
